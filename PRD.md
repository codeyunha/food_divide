# Hi! Pie! — 제품 요구사항 정의서 (PRD)

> 대용량 원재료·완제품 음식을 이웃과 나누는 **소분 파티** 플랫폼
> 문서 버전: v1.0 · 작성일: 2026-07-16

---

## 1. 개요 (Overview)

### 1.1 제품 한 줄 정의
**Hi! Pie!** 는 혼자 다 소비하기 어려운 대용량 원재료 또는 완제품 음식을, 관심 있는 이웃들과 **소분(나눠 사기/나눠 먹기)** 할 수 있도록 파티를 개설하고 참여하는 웹 서비스다.

### 1.2 배경 및 문제 정의
- 코스트코·도매·공동구매 등에서 파는 **대용량 식재료/완제품**은 1인 가구·소규모 가구가 다 쓰기 전에 유통기한이 지나거나 낭비되는 경우가 많다.
- "나눠 사고 싶다"는 니즈는 있으나, 신뢰할 수 있는 상대·투명한 정보(가격·유통기한·용량)·소통 수단이 부족하다.
- 나눠 받은 재료로 **무엇을 해 먹을지** 막막한 경우가 많다.

Hi! Pie! 는 **투명한 파티 정보 등록 + 파티원 전용 채팅 + 재료 기반 레시피 추천**으로 이 문제를 해결한다.

### 1.3 목표 (Goals)
| 구분 | 목표 |
|------|------|
| 핵심 | 소분 파티를 쉽게 개설/참여하고, 파티원끼리 신뢰 있게 소통한다 |
| 차별화 | 등록된 재료를 기반으로 실제 레시피를 추천해 "받은 뒤 활용"까지 연결한다 |
| 신뢰 | 실물·영수증 사진과 명확한 수치(가격/유통기한/용량) 등록을 **필수화**하여 거래 신뢰를 확보한다 |

### 1.4 비목표 (Non-Goals)
- **위치 기반 기능 없음** — 지도, 반경 검색, GPS, 동네 인증 등 위치 정보는 수집·사용하지 않는다.
- 결제/정산 시스템은 v1 범위에서 제외(파티 내 협의로 진행). 향후 로드맵으로 검토.
- 배송/택배 연동 없음.

## 2. 타겟 사용자 (Target Users)

- **규모:** 초기 약 50명 규모의 소규모 커뮤니티(파일럿).
- **연령/특성:** 요리에 관심 있는 **20~40대**.
- **대표 페르소나**
  - **1인 가구 자취생(20대 후반):** 대용량 식재료를 다 못 써서 나눠 사고 싶다.
  - **요리 취미러(30대):** 새로운 재료를 소량 다양하게 써보고 싶고, 레시피 아이디어를 얻고 싶다.
  - **알뜰 가구(40대):** 완제품/반찬을 합리적으로 나눠 구매하고 싶다.

## 3. 핵심 개념 (Core Concepts)

### 3.1 소분 파티(Party)
음식을 나누기 위해 개설하는 단위. 두 종류로 구분한다.

| 유형 | 설명 | 예시 |
|------|------|------|
| **완제품 파티** | 이미 조리·가공된 음식을 나눔 | 대용량 반찬, 냉동만두, 베이커리 세트 |
| **재료 파티** | 조리 전 원재료를 나눔 | 대파 5단, 소고기 3kg, 견과류 대용량 |

### 3.2 역할(Role)
- **파티장(Host):** 파티를 개설하고 정보를 등록·관리, 채팅방을 운영.
- **파티원(Member):** 파티에 참여하고 채팅방에서 소통.

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 파티 개설 (Create Party) — 파티장

파티장이 파티를 개설할 때 아래 항목을 입력한다. **필수 항목이 하나라도 비면 등록 불가.**

| 항목 | 필수 | 설명 |
|------|:---:|------|
| **음식 유형** | ✅ | `완제품` / `원재료(재료)` 중 택 1 |
| **실물 사진** | ✅ | 나눌 음식의 실제 사진 (최소 1장) |
| **영수증 사진** | ✅ | 구매 영수증 사진 — 가격·정품 신뢰 확보용 |
| **음식 태그** | ✅ | 재료/음식 키워드 태그 (예: `#대파` `#소고기`) — **레시피 추천의 입력값이 됨** |
| **가격** | ✅ | 전체 또는 1인분 기준 가격 |
| **유통기한** | ✅ | 소비/유통 기한 날짜 |
| **전체 용량 / 개수** | ✅ | 총 g/kg/ml 또는 개수 |
| **추가 설명** | ⭕ | 음식에 대한 간단한 소개 글(자유 서술) |
| 모집 인원 | ⭕ | 함께 나눌 인원 수 |

**세부 규칙**
- 유형이 `재료`인 경우, 등록된 **태그(재료)** 는 레시피 추천 시스템의 매칭 키로 사용된다.
- 실물 사진과 영수증 사진은 서로 다른 슬롯으로 명확히 구분해 업로드한다.
- 유통기한이 임박/경과한 파티는 목록에서 시각적으로 구분 표기(예: 배지)한다.

### 4.2 파티 탐색 & 참여 (Browse & Join)
- **완제품 파티 / 재료 파티** 목록을 각각 탐색.
- 파티 카드에는 실물 사진, 제목, 태그, 가격, 유통기한, 남은 인원 등이 요약 표시.
- 파티 상세에서 전체 정보(실물·영수증 사진, 수치, 설명)를 확인.
- **참여하기** 버튼 → 파티원으로 합류 + 해당 파티의 **전용 채팅방 자동 참여**.

### 4.3 채팅 (Chat)
- 파티 참여 시, 그 파티원들과 소통 가능한 **채팅방에 참여**.
- 파티 개설 시 **채팅방이 개설**되며, 파티장이 방장이 된다.
- 채팅방은 해당 파티원만 접근 가능(파티 단위 그룹 채팅).
- 기능: 텍스트 메시지 송수신, 새 메시지 알림 배지.

### 4.4 레시피 추천 (Recipe Recommendation) ⭐
- 데이터 소스: **`merged_recipes.json`** (식약처 조리식품 레시피, `COOKRCP01`, 약 1,146건).
- **소분 파티에 등록된 재료(태그) 기반**으로 만들 수 있는 음식을 추천한다.
- 매칭 로직(개요):
  1. 파티의 재료 태그 / 사용자가 보유(참여)한 재료를 입력으로 수집.
  2. 각 레시피의 `RCP_PARTS_DTLS`(재료 상세) 및 `HASH_TAG`(해시태그)와 대조.
  3. 재료 일치도가 높은 순으로 정렬·추천.
- 레시피 상세 노출 항목(데이터 필드 매핑):

| 화면 항목 | JSON 필드 |
|-----------|-----------|
| 요리명 | `RCP_NM` |
| 대표 이미지 | `ATT_FILE_NO_MAIN` / `ATT_FILE_NO_MK` |
| 재료 | `RCP_PARTS_DTLS` |
| 조리 분류 / 방법 | `RCP_PAT2` / `RCP_WAY2` |
| 해시태그 | `HASH_TAG` |
| 영양정보 | `INFO_ENG`(열량) `INFO_CAR`(탄수) `INFO_PRO`(단백) `INFO_FAT`(지방) `INFO_NA`(나트륨) |
| 조리 단계 | `MANUAL01`~`MANUAL20` + `MANUAL_IMG01`~`MANUAL_IMG20` |
| 저감 조리 TIP | `RCP_NA_TIP` |

> 참고: 데이터의 요리명(`RCP_NM`) 등 일부 문자열은 인코딩(EUC-KR/CP949) 보정이 필요할 수 있으므로, 로드 시 **UTF-8 정규화**를 전처리 단계에서 수행한다.

### 4.5 파티 찜하기 (Favorites)
- 관심 있는 파티를 찜(저장)하고, **파티 찜하기** 페이지에서 모아 본다.
- 찜 토글은 카드/상세 어디서든 가능.

### 4.6 내 파티 목록 (My Parties)
- 내가 **개설한 파티** + 내가 **참여한 파티** 를 한 곳에서 관리.
- 각 항목에서 상태(모집중/마감/완료), 채팅방 바로가기 제공.

### 4.7 프로필 (Profile)
- 닉네임, 프로필 이미지, 매너 지표(예: 매너온도), 활동 요약(개설/참여 수).
- 내 정보 수정.

## 6. 데이터 모델 (Data Model, 개념)

### 6.1 Party
```
Party {
  id
  type            // "finished"(완제품) | "ingredient"(재료)
  title
  photos[]        // 실물 사진 (필수, 1+)
  receiptPhoto    // 영수증 사진 (필수)
  tags[]          // 음식/재료 태그 (필수) → 레시피 추천 입력
  price           // 가격 (필수)
  expiryDate      // 유통기한 (필수)
  totalAmount     // 전체 용량(g/kg/ml) 또는 개수 (필수)
  description     // 추가 설명 (선택)
  capacity        // 모집 인원 (선택)
  hostId
  memberIds[]
  chatRoomId
  status          // recruiting | closed | done
  createdAt
}
```

### 6.2 User
```
User { id, nickname, avatar, mannerScore, hostedPartyIds[], joinedPartyIds[], favoritePartyIds[] }
```

### 6.3 ChatRoom / Message
```
ChatRoom { id, partyId, memberIds[] }
Message  { id, roomId, senderId, text, createdAt }
```

### 6.4 Recipe (from merged_recipes.json)
`COOKRCP01.row[]` — 앞의 4.4 필드 매핑 참조. 로드 시 재료 텍스트를 토큰화하여 검색 인덱스 구축.

## 7. 사용자 흐름 (Key User Flows)

**A. 파티 개설(파티장)**
1. `파티 개설` → 유형(완제품/재료) 선택
2. 실물 사진·영수증 사진 업로드 → 태그·가격·유통기한·전체 용량 입력 → 설명 작성
3. 필수값 검증 통과 → 등록 → **채팅방 자동 생성**

**B. 파티 참여(파티원)**
1. `완제품/재료 파티`에서 탐색 → 상세 확인
2. `참여하기` → 파티원 합류 → **채팅방 자동 입장** → 파티장과 협의

**C. 레시피 활용**
1. `레시피` 진입 또는 참여한 재료 파티에서 "이 재료로 만들기"
2. 등록 재료(태그) 기반 추천 목록 → 레시피 상세(재료/조리단계/영양) 확인

## 8. 성공 지표 (Success Metrics, 파일럿)
- 파일럿 50명 중 **파티 1회 이상 개설/참여율**
- 개설된 파티의 **참여 완료(매칭) 비율**
- 필수 정보(사진·수치) **완결 등록률** (신뢰 지표)
- 레시피 추천 **클릭률 / 상세 열람률**

---

## 9. 제약 및 가정 (Constraints & Assumptions)
- **위치 정보 미사용** (수집·저장·표시 없음).
- 데스크톱(PC) 웹 우선 — v5 레이아웃 기준.
- 레시피 데이터는 로컬 정적 파일(`merged_recipes.json`)을 기반으로 하며 서버 API가 아니다(초기).
- 결제/정산은 앱 외 협의 — v1 미포함.

## 10. 향후 로드맵 (Out of Scope for v1)
- 매너온도/평판 시스템 고도화, 랭킹
- 인앱 결제·정산
- 모바일 반응형/앱
- 알림함, 푸시 알림
- 파티 후기·리뷰
